LangChain 概念指南

本指南提供了 LangChain 框架和 AI 应用程序背后关键概念的解释。

我们建议您在深入概念指南之前先学习至少一个教程。这将提供实用的上下文，使您更容易理解这里讨论的概念。

概念指南不涵盖分步说明或特定实现示例 — 这些内容可在操作指南和教程中找到。有关详细参考资料，请参阅 API 参考。

高级概述

  • 架构: LangChain 生态系统中的包组织方式。

核心概念

  • 聊天模型 (Chat models): 通过聊天 API 暴露的 LLM，处理消息序列作为输入并输出一条消息。
  • 消息 (Messages): 聊天模型中的通信单元，用于表示模型输入和输出。
  • 聊天历史 (Chat history): 表示为消息序列的对话，在用户消息和模型响应之间交替。
  • 工具 (Tools): 具有相关模式的函数，定义了函数名称、描述及其接受的参数。
  • 工具调用 (Tool calling): 一种聊天模型 API，接受工具模式和消息作为输入，并在输出消息中返回这些工具的调用。
  • 结构化输出 (Structured output): 使聊天模型以结构化格式（如匹配给定模式的 JSON）响应的技术。
  • 记忆 (Memory): 关于对话的持久化信息，可用于未来对话。
  • 多模态 (Multimodality): 处理不同形式数据（如文本、音频、图像和视频）的能力。
  • 可运行接口 (Runnable interface): 许多 LangChain 组件和 LangChain 表达语言的基础抽象。
  • 流式传输 (Streaming): LangChain 流式 API，用于在生成结果时显示。
  • LangChain 表达语言 (LCEL): 用于编排 LangChain 组件的语法，对简单应用最有用。
  • 文档加载器 (Document loaders): 将来源加载为文档列表。
  • 检索 (Retrieval): 信息检索系统可以响应查询从数据源中检索结构化或非结构化数据。
  • 文本分割器 (Text splitters): 将长文本分割成更小的块，以便单独索引，实现精细检索。
  • 嵌入模型 (Embedding models): 将数据（如文本或图像）表示在向量空间中的模型。
  • 向量存储 (Vector stores): 存储向量及相关元数据并高效搜索。
  • 检索器 (Retriever): 响应查询从知识库返回相关文档的组件。
  • 检索增强生成 (RAG): 通过将语言模型与外部知识库结合来增强它们的技术。
  • 代理 (Agents): 使用语言模型选择要采取的操作序列。代理可以通过工具与外部资源交互。
  • 提示模板 (Prompt templates): 用于分解模型"提示"（通常是消息序列）的静态部分的组件。对于序列化、版本控制和重用这些静态部分很有用。
  • 输出解析器 (Output parsers): 负责接收模型输出并将其转换为更适合下游任务的格式。在工具调用和结构化输出普遍可用之前，输出解析器主要很有用。
  • 少样本提示 (Few-shot prompting): 通过在提示中提供任务的几个示例来提高模型性能的技术。
  • 示例选择器 (Example selectors): 用于根据给定输入从数据集中选择最相关的示例。示例选择器在少样本提示中用于为提示选择示例。
  • 异步编程 (Async programming): 在异步上下文中使用 LangChain 应了解的基础知识。
  • 回调 (Callbacks): 回调允许在内置组件中执行自定义辅助代码。在 LangChain 中，回调用于流式传输 LLM 的输出、跟踪应用程序的中间步骤等。
  • 跟踪 (Tracing): 记录应用程序从输入到输出所经过步骤的过程。跟踪对于调试和诊断复杂应用程序中的问题至关重要。
  • 评估 (Evaluation): 评估 AI 应用程序性能和有效性的过程。包括根据预定义标准或基准测试模型响应，确保其达到所需质量标准并满足预期目的。此过程对于构建可靠的应用程序至关重要。
  • 测试 (Testing): 验证集成或应用程序组件按预期工作的过程。测试对于确保应用程序行为正确且代码库更改不会引入新错误至关重要。

术语表

  • AI 消息块 (AIMessageChunk): AI 消息的部分响应。在从聊天模型流式传输响应时使用。
  • AI 消息 (AIMessage): 表示来自 AI 模型的完整响应。
  • astream_events: 从 LCEL 链中流式传输细粒度信息。
  • 基础工具 (BaseTool): LangChain 中所有工具的基类。
  • 批处理 (batch): 用于使用批量输入执行可运行对象。
  • 绑定工具 (bind_tools): 允许模型与工具交互。
  • 缓存 (Caching): 存储结果以避免对聊天模型的冗余调用。
  • 聊天模型 (Chat models): 处理多种数据模态的聊天模型。
  • 可配置可运行对象 (Configurable runnables): 创建可配置的可运行对象。
  • 上下文窗口 (Context window): 聊天模型可以处理的最大输入大小。
  • 对话模式 (Conversation patterns): 聊天交互中的常见模式。
  • 文档 (Document): LangChain 对文档的表示。
  • 嵌入模型 (Embedding models): 为各种数据类型生成向量嵌入的模型。
  • 人类消息 (HumanMessage): 表示来自人类用户的消息。
  • 注入状态 (InjectedState): 注入到工具函数中的状态。
  • 注入存储 (InjectedStore): 可以注入到工具中用于数据持久化的存储。
  • 注入工具参数 (InjectedToolArg): 向工具函数注入参数的机制。
  • 输入和输出类型 (input and output types): 在可运行对象中用于输入和输出的类型。
  • 集成包 (Integration packages): 与 LangChain 集成的第三方包。
  • 集成测试 (Integration tests): 验证组件之间交互正确性的测试，通常在访问支持集成的底层 API 的情况下运行。
  • 调用 (invoke): 调用可运行对象的标准方法。
  • JSON 模式 (JSON mode): 以 JSON 格式返回响应。
  • langchain-community: LangChain 的社区驱动组件。
  • langchain-core: 核心 langchain 包。包括基础接口和内存中实现。
  • langchain: 用于更高级组件的包（例如，一些预构建的链）。
  • langgraph: LangChain 的强大编排层。用于构建复杂的管道和工作流。
  • langserve: 用于将 LangChain 可运行对象部署为 REST 端点。使用 FastAPI。主要适用于 LangChain 可运行对象，目前不与 LangGraph 集成。
  • LLMs (legacy): 较旧的语言模型，接受字符串作为输入并返回字符串作为输出。
  • 管理聊天历史 (Managing chat history): 维护和管理聊天历史的技术。
  • OpenAI 格式 (OpenAI format): OpenAI 的聊天模型消息格式。
  • RunnableConfig 传播 (Propagation of RunnableConfig): 通过可运行对象传播配置。如果使用 python 3.9、3.10 和异步，请阅读此内容。
  • 速率限制 (rate-limiting): 聊天模型的客户端速率限制。
  • 移除消息 (RemoveMessage): 用于从聊天历史中删除消息的抽象，主要在 LangGraph 中使用。
  • 角色 (role): 表示聊天消息的角色（例如，用户、助手）。
  • 可运行配置 (RunnableConfig): 用于向可运行对象传递运行时信息（例如，run_name、run_id、tags、metadata、max_concurrency、recursion_limit、configurable）。
  • 聊天模型的标准参数 (Standard parameters for chat models): 参数如 API 密钥、temperature 和 max_tokens。
  • 标准测试 (Standard tests): 所有集成必须通过的一组定义的单元和集成测试。
  • 流 (stream): 用于从可运行对象或图表流式传输输出。
  • 标记化 (Tokenization): 将数据转换为标记并反之亦然的过程。
  • 标记 (Tokens): 语言模型在底层读取、处理和生成的基本单位。
  • 工具构件 (Tool artifacts): 向工具输出添加不会发送给模型但可用于下游处理的构件。
  • 工具绑定 (Tool binding): 将工具绑定到模型。
  • @tool: 在 LangChain 中创建工具的装饰器。
  • 工具包 (Toolkits): 可以一起使用的工具集合。
  • 工具消息 (ToolMessage): 表示包含工具执行结果的消息。
  • 单元测试 (Unit tests): 验证单个组件正确性的测试，在隔离环境中运行，不访问互联网。
  • 向量存储 (Vector stores): 专门用于存储和高效搜索向量嵌入的数据存储。
  • with_structured_output: 一个辅助方法，用于原生支持工具调用的聊天模型，以获取匹配通过 Pydantic、JSON 模式或函数指定的给定模式的结构化输出。
  • with_types: 覆盖可运行对象的输入和输出类型的方法。在使用复杂的 LCEL 链并使用 LangServe 部署时很有用。
