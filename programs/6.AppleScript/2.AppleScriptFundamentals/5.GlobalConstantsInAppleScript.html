各项编程语言

Global Constants in AppleScript

AppleScript定义了许多全局常量，您可以在脚本中的任何位置使用它们。

AppleScript Constant

  1. 全局常量 'AppleScript' 提供对您可以在整个脚本中使用的属性的访问

  2. 您可以使用 'AppleScript' 标识符本身来区分AppleScript属性与具有相同名称的当前目标的属性，如节中所示

pi

  1. 这个 “数学” 值表示圆的周长与直径之比
  2. It is defined as a real number with the value 3.14159265359.
  3. For example, the following statement computes the area of a circle with radius 7:
set circleArea to pi * 7 * 7 --result: 153.9380400259

result

  1. 执行语句时，AppleScript将结果值 (如果有) 存储在预定义的属性 “result” 中。
  2. 该值保持在那里，直到执行另一个生成值的语句
  3. 在执行产生结果的语句之前，“结果” 的值是未定义的
  4. 您可以通过在 “脚本” 窗口的 “结果” 窗格中查看 “脚本编辑器” 中的结果.
  5. 注意: 当脚本执行过程中发生错误时，AppleScript会发出错误信号. 它不会在 'result' 属性中返回错误信息. For more information, see AppleScript Error Handling.

Text Constants

  1. AppleScript定义文本属性 space, tab, return, linefeed, and quote.

  2. 您可以有效地将这些属性用作文本常量来表示空格或双引号 ('“') 字符.

  3. 它们在 'text' 类的特殊字符串字符部分中进行了描述.

text item delimiters

  1. AppleScript provides the text item delimiters property for use in processing text.
  2. 此属性包含 一系列 列表strings。 由AppleScript将 列表强制转换为文本或从文本字符串获取文本项时 用作分隔符
  3. When getting text items of text, all of the strings are used as separators. When coercing a list to text, the first item is used as a separator.

Note: Prior to OS X Snow Leopard v10.6, AppleScript only used the first delimiter in the list when getting text items.

  1. Because text item delimiters respect considering and ignoring attributes in AppleScript 2.0, delimiters are case-insensitive by default. Formerly, they were always case-sensitive. To enforce the previous behavior, add an explicit considering case statement.

  2. 获取 text item delimiters属性的值。通常情况下，AppleScript不使用任何分隔符。例如，如果未明确更改文本分隔符，则该语句返回如下结果

{"bread", "milk", "butter", 10.45}  as string

returns the following:

"breadmilkbutter10.45"

For printing or display purposes, it is usually preferable to set text item delimiters to something that’s easier to read. For example, the script

set AppleScript's text item delimiters to {", "}
{"bread", "milk", "butter", 10.45} as string

returns this result:

"bread, milk, butter, 10.45"

“text item delimiters” 属性可用于从路径名中提取单个名称。例如，脚本

set AppleScript's text item delimiters to {":"}
get last text item of "Hard Disk:CD Contents:Release Notes"

returns the result "Release Notes".

如果发生错误，脚本通常使用错误处理程序将 text item delimiters 属性重置为其以前的值 (有关处理错误的更多信息，请参见 [AppleScript错误处理](https://developer.apple.com/library/archive/documentation/AppleScript/Conceptual/AppleScriptLangGuide/conceptual/ASLR_fundamentals.html # // apple_ref/doc/uid/TP40000983-CH218-SW10):

set savedDelimiters to AppleScript's text item delimiters
try
  set AppleScript's text item delimiters to {"**"}
      --other script statements...
      --now reset the text item delimiters:
  set AppleScript's text item delimiters to savedDelimiters
  on error m number n
  --also reset text item delimiters in case of an error:
  set AppleScript's text item delimiters to savedDelimiters
	--and resignal the error:
  error m number n
end try

version

  1. 此属性提供AppleScript的当前版本

  2. 下面的脚本显示了如何检查大于或等于 1.9的版本.

  3. The if statement is wrapped in a considering numeric strings statement

  4. so that an AppleScript version such as 1.10.6 compares as larger than, say, version 1.9.

  5. considering numeric strings
 if version of AppleScript as string ≥ "1.9" then
-- Perform operations that depend on version 1.9 or greater
else
-- Handle case where version is not high enough
end if
end considering

  6. 应用程序可以具有自己的 “版本” 属性，因此要显式访问AppleScript版本，请使用短语 version of AppleScript. 。

  7. 这将在针对另一个应用程序的 “tell” 块中工作，例如:

tell application "Finder"
  version --result: "10.5.1"
  version of AppleScript --result: "2.0"
end tell

current application Constant

  1. current application 常量是指正在执行当前AppleScript脚本的应用程序 (例如，scriptEditor)

  2. 因为当前应用程序是AppleScript的父 (see The AppleScript Inheritance Chain), 它有机会处理当前脚本或AppleScript未处理的命令。

  3. The current application constant is an object specifier—if you ask AppleScript for its value, the result is the object specifier:

get current application --result: current application

However, if you ask for name of current application, AppleScript resolves the object specifier and returns the current application’s name:

name of current application --result: "Script Editor"

missing value Constant

  1. missing value 常量是缺失或未初始化信息的占位符。

  2. For example, the following statements use the missing value constant to determine if a variable has changed:

set myVariable to missing value
if myVariable is equal to missing value then
 -- the value of the variable never changed
 else
 -- the value of the variable did change
end if
