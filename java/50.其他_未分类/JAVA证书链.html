  2. 使用由不在默认信任库中的CA签发的证书

  3. 中间证书未正确配置或丢失

什么是 信任库

在Java中，信任库（TrustStore）是一个存储受信任的证书或证书授权（CA）的库，用于验证和信任连接到的服务器或客户端的身份。信任库中的证书可以是根证书、 中间证书或单独的自签名证书。信任库的作用是在SSL/TLS连接中验证对方的证书，以确保通信的安全性和真实性。

信任库的作用

当Java应用程序使用SSL/TLS协议进行安全通信时，它需要验证对方的证书，以确保通信是与受信任的一方进行的。信任库用于存储这些受信任的证书。如果对方的证书未包含在信任库中，则连接会失败并抛出如你之前看到的 ValidatorException。

信任库的位置

<JAVA_HOME>/lib/security/cacerts

<JAVA_HOME>/jre/lib/security/cacerts

管理信任库

查看内容

keytool -list -v -keystore $JAVA_HOME/lib/security/cacerts -storepass changeit

添加证书到信任库

keytool -import -alias yourAlias -keystore $JAVA_HOME/jre/lib/security/cacerts -file yourCA.crt -storepass changeit

删除信任库的证书

keytool -delete -alias yourAlias -keystore $JAVA_HOME/jre/lib/security/cacerts -storepass changeit

自定义信任库

System.setProperty("javax.net.ssl.trustStore", "path/to/yourTrustStore");
System.setProperty("javax.net.ssl.trustStorePassword", "yourTrustStorePassword");

合并信任库

注意：使用自定义信任库之后，系统默认信任库将不再生效。可以先合并系统默认信任库 到自定义信任库文件

导出

keytool -export -alias defaultAlias -keystore $JAVA_HOME/lib/security/cacerts -file defaultCert.cer -storepass changeit

import javax.net.ssl.*;
import java.security.KeyStore;
import java.io.FileInputStream;

public class CustomTrustStoreExample {
    public static void main(String[] args) {
        try {
            // 加载自定义信任库
            String trustStorePath = "path/to/yourTrustStore";
            String trustStorePassword = "yourTrustStorePassword";
            FileInputStream trustStoreStream = new FileInputStream(trustStorePath);
            
            KeyStore trustStore = KeyStore.getInstance(KeyStore.getDefaultType());
            trustStore.load(trustStoreStream, trustStorePassword.toCharArray());
            
            TrustManagerFactory tmf = TrustManagerFactory.getInstance(TrustManagerFactory.getDefaultAlgorithm());
            tmf.init(trustStore);
            
            SSLContext sslContext = SSLContext.getInstance("TLS");
            sslContext.init(null, tmf.getTrustManagers(), null);
            
            SSLSocketFactory sslSocketFactory = sslContext.getSocketFactory();
            
            // 使用自定义信任库建立SSL连接
            SSLSocket socket = (SSLSocket) sslSocketFactory.createSocket("hostname", 443);
            socket.startHandshake();
            
            System.out.println("SSL connection established using custom TrustStore.");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

  • 别名: www.meizhou.gov.cn，证书的别名。

  • 发布者: 证书的颁发者信息，包含了颁发证书的证书颁发机构 (CA) 的信息，如 Common Name (CN)、Organization (O)、Country (C) 等。

  • 生效时间和失效时间: 证书的有效期开始和结束时间。
  • 证书指纹: 证书的 SHA1 和 SHA256 指纹，用于验证证书的完整性。
  • 签名算法名称: 用于签署该证书的签名算法。
  • 主体公共密钥算法: 证书中使用的公钥算法及其位数。
  • 版本: 证书的版本号。

为什么存在 JAVA无法识别某些域名的证书 而 浏览器能识别

浏览器能更新而 JAVA运行时环境无法实时更新

Java 的默认信任库: Java 应用程序使用的默认信任库是 cacerts 文件，通常位于 $JAVA_HOME/lib/security/cacerts。如果 cacerts 文件中没有包含某个 CA 的证书，那么 Java 应用程序将无法识别该证书。

浏览器的信任库: 浏览器有自己的信任库，通常内置了大量的 CA 根证书，并且会定期更新。这意味着即使某个 CA 证书在 Java 的信任库中缺失，浏览器可能已经更新了其信任库以包括该证书。

证书链

证书链显示了服务器证书和中间证书的详细信息。

s: 表示证书的主体（subject）。

i: 表示证书的颁发者（issuer）。

a: 表示公钥算法和签名算法。

v: 表示证书的有效期。

在这个例子中：

证书 0 是 www.example.org 的证书，由 DigiCert Global G2 TLS RSA SHA256 2020 CA1 签发。
证书 1 是 DigiCert Global G2 TLS RSA SHA256 2020 CA1 的证书，由 DigiCert Global Root G2 签发。

其他例子

s:C=CN, O=Global Digital Cybersecurity Authority Co., Ltd., CN=GDCA TrustAUTH R4 EV SSL CA
i:C=CN, O=GUANG DONG CERTIFICATE AUTHORITY CO.,LTD., CN=GDCA TrustAUTH R5 ROOT
a:PKEY: rsaEncryption, 2048 (bit); sigalg: RSA-SHA256
v:NotBefore: Apr  6 03:35:09 2016 GMT; NotAfter: Dec 30 16:00:00 2030 GMT
   
s:C=CN, O=Global Digital Cybersecurity Authority Co., Ltd., CN=GDCA TrustAUTH R4 EV SSL CA：


s 代表证书的主题 (Subject)，也就是该证书所代表的实体。
C=CN：国家（China，中国）。
O=Global Digital Cybersecurity Authority Co., Ltd.：组织（全球数字网络安全认证管理有限公司）。
CN=GDCA TrustAUTH R4 EV SSL CA：证书的公共名称（CN），即证书的具体标识名称（GDCA TrustAUTH R4 EV SSL CA）。
i:C=CN, O=GUANG DONG CERTIFICATE AUTHORITY CO.,LTD., CN=GDCA TrustAUTH R5 ROOT：

i 代表证书的颁发者（Issuer），即签发该证书的实体。
C=CN：国家（China，中国）。
O=GUANG DONG CERTIFICATE AUTHORITY CO.,LTD.：组织（广东省证书认证有限公司）。
CN=GDCA TrustAUTH R5 ROOT：证书的公共名称（CN），即根证书的具体标识名称（GDCA TrustAUTH R5 ROOT）。
a:PKEY: rsaEncryption, 2048 (bit); sigalg: RSA-SHA256：

PKEY 表示证书使用的公钥算法，这里是 RSA 加密算法，密钥长度为2048位。
sigalg 表示证书签名算法，这里是 RSA-SHA256。
v:NotBefore: Apr 6 03:35:09 2016 GMT; NotAfter: Dec 30 16:00:00 2030 GMT：

NotBefore 和 NotAfter 表示证书的有效期，NotBefore 是证书生效的开始时间，NotAfter 是证书过期的时间。
在这个例子中，证书的有效期从 2016年4月6日03:35:09 GMT 开始，到 2030年12月30日16:00:00 GMT 结束。

如何判断一个CA机构 是 根机构 还是 中间机构

判断它的证书是 自签名证书 还是 其他机构颁发的证书

openssl x509 -in  GDCA\ TrustAUTH\ R5\ ROOT.cer -noout -issuer -subject

issuer=C=CN, O=GUANG DONG CERTIFICATE AUTHORITY CO.,LTD., CN=GDCA TrustAUTH R5 ROOT
subject=C=CN, O=GUANG DONG CERTIFICATE AUTHORITY CO.,LTD., CN=GDCA TrustAUTH R5 ROOT

如果颁发者 跟 对象 一致，则说明是 自签名证书，说明是跟机构（root CA ）。如果 颁发者 跟 对象不一致 则说明是 中间机构

给JAVA配置跟证书

keytool -import -alias GDCA\ TrustAUTH\ R5\ ROOT -file GDCA\ TrustAUTH\ R5\ ROOT.cer -keystore $JAVA_HOME/jre/lib/security/cacerts -storepass changeit
