RUST学习笔记

RefCell<T> 和 内部可变性模式

  1. 内部可变性（Interior mutability）是 Rust 中的一个设计模式

  2. 它允许你即使在有不可变引用时也可以改变数据，这通常是借用规则所不允许的

  3. 为了改变数据，该模式在数据结构中使用 unsafe 代码来模糊 Rust 通常的可变性和借用规则

  4. 我们还未讲到不安全代码；第十九章会学习它们

  5. 当可以确保代码在运行时会遵守借用规则，即使编译器不能保证的情况，可以选择使用那些运用内部可变性模式的类型

  6. 所涉及的 unsafe 代码将被封装进安全的 API 中，而外部类型仍然是不可变的。

通过 RefCell<T> 在运行时检查借用规则

如下为选择 Box<T>，Rc<T> 或 RefCell<T> 的理由：

  • Rc<T> 允许相同数据有多个所有者；Box<T> 和 RefCell<T> 有单一所有者。
  • Box<T> 允许在编译时执行不可变或可变借用检查；Rc<T>仅允许在编译时执行不可变借用检查；RefCell<T> 允许在运行时执行不可变或可变借用检查。
  • 因为 RefCell<T> 允许在运行时执行可变借用检查，所以我们可以在即便 RefCell<T> 自身是不可变的情况下修改其内部的值。

在不可变值内部改变值就是 内部可变性 模式。让我们看看何时内部可变性是有用的，并讨论这是如何成为可能的。

内部可变性：不可变值的可变借用

  1. 借用规则的一个后果是，当你拥有一个不可变值时，你不能以可变的方式借用它。例如，这段代码将无法编译：
fn main() {
    let x = 5;
    let y = &mut x;
}

  1. 然而，在某些情况下，如果一个值能在其方法中自我变更，同时对其他代码保持不可变的外观，这会非常有用。

  2. 值之外的代码将无法改变这个值。使用RefCell<T>是获得内部可变性的一种方式，但RefCell<T>并不能完全绕过借用规则：

    1. 编译器中的借用检查器允许这种内部可变性，并且借用规则会在运行时进行检查。如果你违反了这些规则，你将得到一个运行时错误而不是编译错误。

让我们通过一个实际的例子来看看我们如何使用RefCell<T>来改变一个不可变的值，以及为什么这样做是有用的。

内部可变性示例

A Use Case for Interior Mutability: Mock Objects

  1. 我们将创建一个库，这个库用于跟踪当前值 与 最大值的接近程度，不同的接近程度，发出不同的消息。（例如这个库可以用于 跟踪用户对API调用次数的消耗）

  2. 我们的库只提供跟踪值与最大值接近程度的功能，以及在什么时候应该发送什么消息

  3. 代码示例

#![allow(unused)]
fn main() {
#[test]
pub fn send_message() {
    let mut tracker = Tracker::new(crate::tests::MockMessageSender::new(), 100);
    tracker.set(76);
    assert_eq!(tracker.sender.sent_messages.len(), 1);
    tracker.set(90);
    assert_eq!(tracker.sender.sent_messages.len(), 2);
    tracker.set(100);
    assert_eq!(tracker.sender.sent_messages.len(), 3);
}

struct MockMessageSender {
    sent_messages: Vec<String>,
}

impl MockMessageSender {
    fn new() -> MockMessageSender {
        MockMessageSender {
            sent_messages: vec![],
        }
    }
}

impl MessageSender for MockMessageSender {
    fn send_message(&mut self, message: &str) {
        self.sent_messages.push(message.to_string());
    }
}

trait MessageSender {
    fn send_message(&mut self, message: &str);
}

struct Tracker<T: MessageSender> {
    sender: T,
    value: usize,
    max: usize,
}

impl<T: MessageSender> Tracker<T> {
    fn new(sender: T, max: usize) -> Tracker<T> {
        Tracker {
            sender,
            value: 0,
            max,
        }
    }
    fn set(&mut self, value: usize) {
        self.value = value;
        let i = self.value as f64 / self.max as f64;
        if i >= 1.0 {
            self.sender.send_message("Error: You are over the limit!");
        } else if i >= 0.9 {
            self.sender.send_message("Warning: You've used up over 90% of your quota");
        } else if i >= 0.75 {
            self.sender.send_message("Warning: You've used up over 75% of your quota");
        }
    }
}
}

上述代码中，实现了一个Tracker。以及一个 Mock MessageSender。但是上述代码有一个问题

MessageSender 对外暴露了 可变引用。使得客户端可以随意修改内边可变变量。

而这就到了 RefCell的使用场景了。 RefCell使得 不可变引用 能够 的借用。

修改后的代码如下

#![allow(unused)]
fn main() {
#[test]
pub fn send_message() {
    let mut tracker = Tracker::new(crate::tests::MockMessageSender::new(), 100);
    tracker.set(76);
    assert_eq!(tracker.sender.sent_messages.borrow().len(), 1);
    tracker.set(90);
    assert_eq!(tracker.sender.sent_messages.borrow().len(), 2);
    tracker.set(100);
    assert_eq!(tracker.sender.sent_messages.borrow().len(), 3);
}

struct MockMessageSender {
    sent_messages: RefCell<Vec<String>>,
}

impl MockMessageSender {
    fn new() -> MockMessageSender {
        MockMessageSender {
            sent_messages: RefCell::new(vec![])
        }
    }
}

impl MessageSender for MockMessageSender {
    fn send_message(& self, message: &str) {
        self.sent_messages.borrow_mut().push(message.to_string());
    }
}

trait MessageSender {
    fn send_message(&self, message: &str);
}

struct Tracker<T: MessageSender> {
    sender: T,
    value: usize,
    max: usize,
}

impl<T: MessageSender> Tracker<T> {
    fn new(sender: T, max: usize) -> Tracker<T> {
        Tracker {
            sender,
            value: 0,
            max,
        }
    }
    fn set(&mut self, value: usize) {
        self.value = value;
        let i = self.value as f64 / self.max as f64;
        if i >= 1.0 {
            self.sender.send_message("Error: You are over the limit!");
        } else if i >= 0.9 {
            self.sender.send_message("Warning: You've used up over 90% of your quota");
        } else if i >= 0.75 {
            self.sender.send_message("Warning: You've used up over 75% of your quota");
        }
    }
}
}

RefCell能够使得 我们在 结构体内部 通过不可变 引用 封装 可变性方法

#![allow(unused)]
fn main() {
let vec1 = vec![1, 2, 3];
let cell = RefCell::new(vec1);

{
    let mut ref_mut1 = cell.borrow_mut();
    ref_mut1.push(4);
    println!("{:?}", ref_mut1);
}
let mut ref_mut2= cell.borrow_mut();
}

RefCell<T> 在运行时记录借用

  • 当创建不可变和可变引用时，我们分别使用 & 和 &mut 语法。对于 RefCell<T> 来说，则是 borrow 和 borrow_mut 方法，这属于 RefCell<T> 安全 API 的一部分
  • borrow 方法返回 Ref<T> 类型的智能指针，borrow_mut 方法返回 RefMut 类型的智能指针
  • 这两个类型都实现了 Deref，所以可以当作常规引用对待。
  • RefCell<T> 记录当前有多少个活动的 Ref<T> 和 RefMut<T> 智能指针
  • 每次调用 borrow，RefCell<T> 将活动的不可变借用计数加一，当 Ref<T> 值离开作用域时，不可变借用计数减一，就像编译时借用规则一样
  • RefCell<T> 在任何时候只允许有多个不可变借用或一个可变借用。
  • 如果我们尝试违反这些规则，相比引用时的编译时错误，RefCell<T> 的实现会在运行时出现 panic

结合 Rc<T> 和 RefCell<T> 来拥有多个可变数据所有者

  • RefCell<T> 的一个常见用法是与 Rc<T> 结合
  • Rc<T> 允许对相同数据有多个所有者，不过只能提供数据的不可变访问
  • 如果有一个储存了 RefCell<T> 的 Rc<T> 的话，就可以得到有多个所有者 并且 可以修改的值了！
#![allow(unused)]
fn main() {
pub fn test_ref_rc(){
    let vec1 = vec![1, 2, 3];
    let cell = RefCell::new(vec1);
    let rc = Rc::new(cell);
    let rc1 = rc.clone();
    let rc2 = rc.clone();
    rc1.borrow_mut().push(4);
    rc2.borrow_mut().push(5);

    println!("{:?}",rc);
}
}

总结

RefCell 将 引用规则（可以存在多个不可变引用、或者一个可变引用） 从 编译时 转移到 运行时。
