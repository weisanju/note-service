RUST学习笔记

Rc<T> 引用计数智能指针

  1. 大部分情况下所有权是非常明确的:可以准确地知道哪个变量拥有某个值
  2. 有些情况单个值可能会有多个所有者,例如图数据结构。一个节点 不应该被清理，除非没有任何边 在向它
  3. 为了启用多所有权，Rust 有一个叫做 Rc<T> 的类型。其名称为 引用计数（reference counting）的缩写
  4. 引用计数意味着记录一个值引用的数量来知晓这个值是否仍在被使用。如果某个值有零个引用，就代表没有任何有效引用并可以被清理。
  5. Rc<T> 用于当我们希望在堆上分配一些内存供程序的多个部分读取，而且无法在编译时确定程序的哪一部分会最后结束使用它的时候，如果确实知道哪部分是最后一个结束使用的话，就可以令其成为数据的所有者，正常的所有权规则就可以在编译时生效。

注意 Rc<T> 只能用于单线程场景

使用 Rc<T> 共享数据

  1. 不必像调用 Rc::clone 增加引用计数那样调用一个函数来减少计数；

  2. Drop trait 的实现当 Rc<T> 值离开作用域时自动减少引用计数。

  3. 使用 Rc<T> 允许一个值有多个所有者，引用计数则确保只要任何所有者依然存在其值也保持有效。

  4. 通过不可变引用， Rc<T> 允许在程序的多个部分之间只读地共享数据。如果 Rc<T> 也允许多个可变引用,则会违反第四章讨论的借用规则之一：相同位置的多个可变借用可能造成数据竞争和不一致

使用Box引用无法共享数据

Box只能允许在编译时期无固定大小

#![allow(unused)]

fn main() {
    enum List {
        Cons(i32, Box<List>),
        Nil,
    }
    #[test]
    fn test_data_share_1(){
        let a = Cons(5, Box::new(Cons(10, Box::new(Nil))));

        let b = Cons(3, Box::new(a));

        let c = Cons(4, Box::new(a));
    }
}

使用RC共享数据

enum List {
    Cons(i32, Rc<List>),
    Nil,
}

use crate::List::{Cons, Nil};
use std::rc::Rc;

fn main() {
    let a = Rc::new(Cons(5, Rc::new(Cons(10, Rc::new(Nil)))));
    let b = Cons(3, Rc::clone(&a));
    let c = Cons(4, Rc::clone(&a));
}

也可以调用 a.clone 拷贝，但是根据rust惯例 调用 Rc::clone 是浅拷贝，仅仅增加了一次引用次数。a.clone是深拷贝

Cloning an Rc Increases the Reference Count

#![allow(unused)]
fn main() {
    #[test]
    fn test_data_share_2() {
        let a = Rc::new(List1::Cons(5, Rc::new(List1::Cons(10, Rc::new(List1::Nil)))));
        println!("count after creating a = {}", Rc::strong_count(&a));
        let b = List1::Cons(3, Rc::clone(&a));
        println!("count after creating b = {}", Rc::strong_count(&a));
        let c = List1::Cons(4, Rc::clone(&a));

        println!("count after creating c = {}", Rc::strong_count(&a));
        drop(b);
        println!("count after drop b = {}", Rc::strong_count(&a));
    }
}

多个部分之间共享多个不可变引用

通过不可变引用，Rc<T>允许你在程序的多个部分之间共享数据，但仅限于只读。如果Rc<T>也允许你拥有多个可变引用，那么可能会违反第4章讨论的借用规则之一：对同一位置进行多个可变借用可能导致数据竞争和不一致性。但能够改变数据非常有用！在下一节中，我们将讨论内部可变性模式和RefCell<T>类型，你可以将其与Rc<T>一起使用以解决这种不可变性限制。
