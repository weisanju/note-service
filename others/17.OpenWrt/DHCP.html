DNS and DHCP configuration /etc/config/dhcp

See also: DNS and DHCP examples, dnsmasq, odhcpd

OpenWrt使用dnsmasq和odhcpd作为默认的DNS/DHCP和DHCPv6服务器。

dnsmasq是一个轻量级的DNS和DHCP服务器软件。在OpenWrt中，它被用作默认的DNS和DHCP服务器。dnsmasq可以解析域名并将其映射到相应的IP地址，同时还可以为网络上的设备分配IP地址。

odhcpd是一个用于IPv6网络的DHCPv6服务器。DHCPv6是一种IPv6网络中自动分配IP地址和其他网络配置的协议。odhcpd在OpenWrt中被用作默认的DHCPv6服务器，它负责为支持IPv6的设备分配IPv6地址和其他必要的网络配置。

通过使用dnsmasq和odhcpd作为默认的DNS/DHCP和DHCPv6服务器，OpenWrt能够提供可靠的网络服务和灵活的网络配置选项。用户可以根据自己的需求进行定制和优化。

Dnsmasq作为一个下游缓存DNS服务器的功能，它会向DHCP客户端广告自己，从而在本地网络上提供更好的DNS功能性能和管理。当Dnsmasq收到一个目前不在缓存中的DNS查询请求时，它会将该请求转发给上游DNS服务器。这样可以减少DNS查询的延迟和网络流量，并提高DNS查询的可靠性。

Sections

这段内容解释了dhcp配置文件中可能出现的部分类型。并非所有类型都会在文件中出现，大多数类型只适用于特殊配置。常见的类型包括：常规选项(Common Options)、DHCP池(DHCP Pools)和静态租约(Static Leases)。

默认配置包含一个常规部分，用于指定DNS和守护程序相关选项，以及一个或多个DHCP池，用于定义网络接口上的DHCP服务。

Common options

这段内容解释了“dnsmasq”类型的部分在配置文件中的作用，它可以指定每个“dnsmasq”实例的值和选项，包括所有接口所服务的DHCP选项。该表列出了所有可用的选项、它们的默认值以及相应的“dnsmasq”命令行选项。需要查看更多详细信息可以参考“dnsmasq”手册。

这些是常用选项的默认设置:

# uci -N show dhcp.@dnsmasq[0]
dhcp.@dnsmasq[0]=dnsmasq
dhcp.@dnsmasq[0].domainneeded='1'
dhcp.@dnsmasq[0].boguspriv='1'
dhcp.@dnsmasq[0].filterwin2k='0'
dhcp.@dnsmasq[0].localise_queries='1'
dhcp.@dnsmasq[0].rebind_protection='1'
dhcp.@dnsmasq[0].rebind_localhost='1'
dhcp.@dnsmasq[0].local='/lan/'
dhcp.@dnsmasq[0].domain='lan'
dhcp.@dnsmasq[0].expandhosts='1'
dhcp.@dnsmasq[0].nonegcache='0'
dhcp.@dnsmasq[0].authoritative='1'
dhcp.@dnsmasq[0].readethers='1'
dhcp.@dnsmasq[0].leasefile='/tmp/dhcp.leases'
dhcp.@dnsmasq[0].resolvfile='/tmp/resolv.conf.d/resolv.conf.auto'
dhcp.@dnsmasq[0].nonwildcard='1'
dhcp.@dnsmasq[0].localservice='1'
  • local and domain enable dnsmasq to serve entries in /etc/hosts, as well as DHCP client's names if configured under lan domain.
  • domainneeded, boguspriv, localise_queries, and expandhosts ensure requests for local host names are not forwarded to upstream DNS servers.
  • authoritative makes router the only DHCP server on this network; clients get their IP lease a lot faster this way.
  • leasefile stores leases in a file so they can be picked up again if dnsmasq is restarted.
  • resolvfile tells dnsmasq to use this file to find upstream name servers; it gets created by the WAN DHCP or PPP client.
  • enable_tftp and tftp_root turn on the TFTP server and serve files from tftp_root.
    • May need to set server's IP on client, changing it by setting serverip (e.g. setenv serverip 192.168.1.10).

这段内容是关于dnsmasq配置的解释。dnsmasq是一个轻量级的DNS和DHCP服务器软件。下面是对每个配置选项的解释：

  • local和domain使得dnsmasq能够提供/etc/hosts中的条目，以及在lan域配置的DHCP客户端的名称。
  • domainneeded、boguspriv、localise_queries和expandhosts确保本地主机名的请求不会转发到上游DNS服务器。
  • authoritative使得路由器成为该网络上唯一的DHCP服务器；这样可以更快地为客户端提供IP租约。
  • leasefile将租约存储在一个文件中，这样如果dnsmasq重新启动，可以重新获取租约。
  • resolvfile告诉dnsmasq使用该文件来查找上游名称服务器；它由WAN DHCP或PPP客户端创建。
  • enable_tftp和tftp_root打开TFTP服务器，并从tftp_root目录提供文件。
  • 可能需要在客户端上设置服务器的IP，通过设置serverip来更改它（例如，setenv serverip 192.168.1.10）。
