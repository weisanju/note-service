Python logging库使用指南

1. 基本概念

Python的logging库是标准库的一部分，提供了灵活的日志记录功能。它允许开发者记录程序运行时的各种信息，便于调试和监控。

1.1 日志级别

logging库定义了以下几个日志级别（按严重程度递增）：

  • DEBUG (10): 详细信息，通常用于调试问题
  • INFO (20): 确认程序按预期运行
  • WARNING (30): 表示可能出现的问题，程序仍在正常工作
  • ERROR (40): 由于更严重的问题，程序无法执行某些功能
  • CRITICAL (50): 严重错误，表明程序可能无法继续运行

默认情况下，logging模块只显示WARNING及以上级别的日志。

2. 基本使用

2.1 简单示例

import logging

# 基本配置
logging.basicConfig(level=logging.INFO)

# 记录不同级别的日志
logging.debug("这是一条调试信息")  # 不会显示，因为级别设置为INFO
logging.info("这是一条信息")
logging.warning("这是一条警告")
logging.error("这是一条错误信息")
logging.critical("这是一条严重错误信息")

2.2 配置日志格式

import logging

# 配置日志格式
logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    datefmt='%Y-%m-%d %H:%M:%S'
)

logging.debug("带有详细时间格式的调试信息")

3. 将日志写入文件

3.1 基本文件日志

import logging

# 配置日志写入文件
logging.basicConfig(
    filename='app.log',  # 日志文件名
    filemode='w',  # 'w'表示覆盖，'a'表示追加
    level=logging.DEBUG,
    format='%(asctime)s - %(levelname)s - %(message)s'
)

logging.debug("这条信息将写入文件")

3.2 同时输出到控制台和文件

import logging
import sys

# 创建logger
logger = logging.getLogger()
logger.setLevel(logging.DEBUG)

# 创建文件处理器
file_handler = logging.FileHandler('app.log')
file_handler.setLevel(logging.DEBUG)

# 创建控制台处理器
console_handler = logging.StreamHandler(sys.stdout)
console_handler.setLevel(logging.INFO)

# 创建格式器
formatter = logging.Formatter('%(asctime)s - %(name)s - %(levelname)s - %(message)s')
file_handler.setFormatter(formatter)
console_handler.setFormatter(formatter)

# 添加处理器到logger
logger.addHandler(file_handler)
logger.addHandler(console_handler)

# 记录日志
logger.debug("这条信息将写入文件")
logger.info("这条信息将同时显示在控制台和写入文件")

4. 高级配置

4.1 使用配置文件

可以使用配置文件来配置logging，例如logging.conf：

[loggers]
keys=root,sampleLogger

[handlers]
keys=consoleHandler,fileHandler

[formatters]
keys=sampleFormatter

[logger_root]
level=DEBUG
handlers=consoleHandler,fileHandler

[logger_sampleLogger]
level=DEBUG
handlers=consoleHandler,fileHandler
qualname=sampleLogger
propagate=0

[handler_consoleHandler]
class=StreamHandler
level=INFO
formatter=sampleFormatter
args=(sys.stdout,)

[handler_fileHandler]
class=FileHandler
level=DEBUG
formatter=sampleFormatter
args=('app.log', 'a')

[formatter_sampleFormatter]
format=%(asctime)s - %(name)s - %(levelname)s - %(message)s
datefmt=%Y-%m-%d %H:%M:%S

然后在代码中使用：

import logging
import logging.config

# 加载配置文件
logging.config.fileConfig('logging.conf')

# 获取logger
logger = logging.getLogger('sampleLogger')

# 记录日志
logger.debug('这是一条调试信息')

4.2 使用字典配置

import logging
import logging.config

# 配置字典
config = {
    'version': 1,
    'formatters': {
        'standard': {
            'format': '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
        },
    },
    'handlers': {
        'console': {
            'class': 'logging.StreamHandler',
            'level': 'INFO',
            'formatter': 'standard',
            'stream': 'ext://sys.stdout'
        },
        'file': {
            'class': 'logging.FileHandler',
            'level': 'DEBUG',
            'formatter': 'standard',
            'filename': 'app.log',
            'mode': 'a',
        }
    },
    'loggers': {
        '': {  # root logger
            'handlers': ['console', 'file'],
            'level': 'DEBUG',
            'propagate': True
        }
    }
}

# 应用配置
logging.config.dictConfig(config)

# 获取logger
logger = logging.getLogger(__name__)

# 记录日志
logger.debug('这是一条调试信息')
logger.info('这是一条信息')

5. 日志轮转

对于长期运行的应用程序，日志文件可能会变得非常大。可以使用RotatingFileHandler或TimedRotatingFileHandler来管理日志文件的大小和数量。

5.1 基于大小的日志轮转

import logging
from logging.handlers import RotatingFileHandler

# 创建logger
logger = logging.getLogger()
logger.setLevel(logging.DEBUG)

# 创建轮转文件处理器
handler = RotatingFileHandler(
    'app.log',
    maxBytes=10*1024*1024,  # 10MB
    backupCount=5  # 保留5个备份文件
)
handler.setLevel(logging.DEBUG)

# 设置格式
formatter = logging.Formatter('%(asctime)s - %(name)s - %(levelname)s - %(message)s')
handler.setFormatter(formatter)

# 添加处理器
logger.addHandler(handler)

# 记录日志
logger.debug('这是一条调试信息')

5.2 基于时间的日志轮转

import logging
from logging.handlers import TimedRotatingFileHandler

# 创建logger
logger = logging.getLogger()
logger.setLevel(logging.DEBUG)

# 创建时间轮转文件处理器
handler = TimedRotatingFileHandler(
    'app.log',
    when='midnight',  # 每天午夜轮转
    interval=1,  # 每1个单位时间轮转一次
    backupCount=7  # 保留7个备份文件
)
handler.setLevel(logging.DEBUG)

# 设置格式
formatter = logging.Formatter('%(asctime)s - %(name)s - %(levelname)s - %(message)s')
handler.setFormatter(formatter)

# 添加处理器
logger.addHandler(handler)

# 记录日志
logger.debug('这是一条调试信息')

6. 最佳实践

  1. 使用模块级别的logger：每个模块使用自己的logger，便于定位日志来源

    logger = logging.getLogger(__name__)

  2. 适当设置日志级别：开发环境可以使用DEBUG级别，生产环境使用INFO或WARNING级别

  3. 结构化日志信息：使用格式化字符串提供上下文信息

    logger.error("处理用户 %s 的请求时出错: %s", user_id, error_message)

  4. 捕获异常时记录堆栈跟踪：

    try:
    # 可能引发异常的代码
    result = 10 / 0
except Exception as e:
    logger.exception("发生异常: %s", str(e))  # 自动包含堆栈跟踪

  5. 配置日志轮转：防止日志文件过大

  6. 在多线程环境中使用线程安全的处理器

7. 常见问题解决

  1. 重复日志：可能是由于多次调用basicConfig或添加了多个相同的处理器

    • 解决方法：确保只配置一次，或在添加处理器前检查是否已存在

  2. 日志不显示：可能是日志级别设置过高

    • 解决方法：检查logger和handler的级别设置

  3. 性能问题：频繁的日志记录可能影响性能

    • 解决方法：使用logger.isEnabledFor()检查日志级别，避免不必要的字符串格式化

8. 总结

Python的logging库提供了灵活而强大的日志记录功能，适用于从简单脚本到复杂应用程序的各种场景。通过合理配置，可以实现日志的分级记录、格式化输出、文件存储和轮转等功能，帮助开发者更好地监控和调试程序。
