SSH代理

常用选项

1 | -C 压缩传输，加快传输速度 2 | -f 在后台对用户名密码进行认证 3 | -N 仅仅只用来转发，不用再弹回一个新的shell -n 后台运行 4 | -q 安静模式，不要显示任何debug信息 5 | -l 指定ssh登录名 6 | -g 允许远程主机连接到本地用于转发的端口 7 | -L 进行本地端口转发 8 | -R 进行远程端口转发 9 | -D 动态转发，即socks代理 10 | -T 禁止分配伪终端 11 | -p 指定远程ssh服务端口

A -> B -> C

本地端口代理

将本地主机A的端口8888 由远程主机B 转发到目标主机C的IP地址和端口上。 ssh -L 8888:target-host:target-port proxy-host-user@proxy-host -p proxy-host-port

f:后台运行 q:安静模式 N:不执行远程命令

ssh -fqN -L 8888:192.168.3.11:80 weisanju@weisanju.dynv6.net -p 65533 ssh -fqN -L 8888:192.168.3.24:5230 weisanju@weisanju.dynv6.net -p 65533

远程端口代理

  1. 在代理主机 proxy-host 上开启一个 access-port 端口 监听。
  2. 并且代理主机将该端口的数据转发到目标主机 target-host 的 target-port 端口上。
  3. 执行此命令的方是跳板机或者被代理主机：这条命令可以在被代理的主机上执行、也可以在代理的主机上执行
  4. 注意：代理主机上的sshd_config需要开启 gatewayports yes 表示开启的端口监听在所有网卡上
ssh -R <proxy-host-access-port>:target-host:target-port proxy-host-user@proxy-host -p proxy-host-port

动态端口代理

可以理解为：在执行命令的地方 启动一个 socks5代理。将数据转发到代理主机上，由代理主机转发到目标主机上。

ssh -D <local-port> proxy-host-user@proxy-host -p proxy-host-port

SSH自动重连

autossh -M 0 -fqN -L 8888:192.168.3.24:5230 weisanju@weisanju.dynv6.net -p 65533
autossh -M 0 -fqN -L 8889:192.168.3.2:80 weisanju@weisanju.dynv6.net -p 65533

autossh -M 0 -fqN -L 8889:192.168.3.1:80 weisanju@weisanju.dynv6.net -p 65533

其中，-M 0表示关闭autossh的监控端口，-f表示在后台运行，-N表示不执行远程命令，只建立SSH连接，-L表示本地端口转发，<本地端口> 是你要监听的本地端口，<目标主机>和<目标端口>是你要访问的目标主机和端口，<SSH用户名>和<SSH服务器地址>是你的SSH登录信息。

SSH配置自定义主机

Host <自定义主机名> HostName <主机IP或域名> Port <端口号> User <用户名>

ProxyCommand

ProxyCommand：用来指定连接到服务器的命令. 其可以是任何的命令,只要能从其标准输入读入数据,然后写出到标准输出即可. 这条命令需要连接到sshd服务器上.

ssh -W命令 -W host:port：将client过来的标准输入和输出forward到host和port指定的地方. 可以看到,这个选项直接就可以搭配上ProxyCommand的需求.

单层跳板机

Host 跳板机
    HostName <主机IP或域名>
    Port <端口号>
    User <用户名>
    IdentityFile <私钥路径>
    StrictHostKeyChecking no
    UserKnownHostsFile /dev/null

Host <自定义主机名>
    HostName <主机IP或域名>
    Port <端口号>
    User <用户名>
    ProxyCommand ssh -q -x -W %h:%p <跳板机>
